많이 사용하는 데에는 이유가 있다.. java 문법 자체의 verbose한 점과 복잡하고 오래된 기술이라는 인식이 강했다. 근데 FastAPI로 프로젝트를 진행하면서 왜 Spring을 채택하는 기업이 많은지 알 것 같다. 
Spring은 개발자에게 강제하는 점이 많다. Model View Controller를 쪼개고 Service 레이어를 만들어서 비즈니스 로직을 따로 만들며, Repository를 따로 만들어서 db 입출력을 관리한다. 심지어 인터페이스로 Repository를 갈아끼울 수 있다... FastAPI같은 것에서는 강제하지 않는 행위이다. 그게 편할지 몰라도 프로젝트가 좀만 커지면 너무 불편해짐을 알 수 있다.
심지어 빈이라는 것으로 싱글턴으로 객체를 관리해주다니.. 일단 첫인상은 좋다.
